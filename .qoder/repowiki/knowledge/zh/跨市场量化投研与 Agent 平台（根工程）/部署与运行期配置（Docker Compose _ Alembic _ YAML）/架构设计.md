模块按职责分为三层：
- `deploy/`：以 `docker-compose.yml` 为入口，编排 PostgreSQL 16、Redis 7、MinIO S3 兼容存储、MLflow v2.14.3（后端存 PG、工件存 MinIO）、Prometheus 与 Grafana 六个服务，通过 `.env.example` 暴露环境变量模板，所有敏感值经 `${VAR:-default}` 注入。
- `configs/`：YAML 分层配置，`base.yaml` 定义市场、风险引擎、模型注册表、MCP 白名单等全局默认；`dev.yaml` 使用 `extends: base.yaml` 覆盖并引入 `DATABASE_URL`、`REDIS_URL`、S3 端点及 MLflow tracking URI 等运行时变量，形成“基线 + 环境覆盖”的加载顺序。
- `sql/migrations/`：Alembic 在线迁移目录，`env.py` 显式禁用 autogenerate（注释强调安全相关表必须手写），通过 `DATABASE_URL` 环境变量驱动连接；`versions/` 下每个文件对应一个幂等的 `upgrade()/downgrade()` 对，文件名采用 `YYYYMMDD_NNNN_name.py` 时间戳+序号前缀，revision 用短字符串而非 UUID。
依赖方向：应用代码读取 `configs/*.yaml` 并通过环境变量注入；`deploy/docker-compose.yml` 仅负责拉起外部依赖，不耦合业务逻辑。
