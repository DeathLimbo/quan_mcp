顶层 `pyproject.toml` 通过 `hatchling.build.targets.wheel.packages = ["packages", "apps"]` 将 `packages/*`（领域层：common/instrument/ingestion/features/training/portfolio/risk 等）与 `apps/*`（FastAPI / RQ Worker / quant-read-mcp / quant-admin-mcp / scheduler）打包为同一可分发项目，子模块仅通过公开 Python 接口互相调用，禁止跨目录直接 import。

- 运行时由 `deploy/docker-compose.yml` 拉起 PostgreSQL + Redis + MinIO，`alembic.ini` 指向 `sql/migrations`，所有子应用共享同一套连接配置。
- `README.md` 的「顶层约束」（UTC 时间轴、InstrumentId 唯一键、Point-in-time 查询、一份特征代码、Fail-closed、禁令）作为跨 apps 与 packages 的共同契约，被 tests/golden dataset 与 MCP Skill 校验共同遵守。
- 测试套件位于 `tests/`，按 `pytest.mark.integration/replay/golden` 标记区分依赖 docker-compose 栈、历史回放与黄金回归三类场景。
