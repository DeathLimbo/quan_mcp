本模块是一个 LLM Skill 定义，由顶层 SKILL.md 驱动，内部按职责分为三层：
- 规则层：SKILL.md 声明触发条件、绝对约束（禁止捏造数据、fail-closed、PIT 纪律、只读）、标准五步工作流（resolve identity → fetch bars → score → risk_check → report），以及可调用工具清单。
- 引用层：references/ 存放不可变参考文档，包括 InstrumentId 规范（`{market}.{venue}.{asset_type}.{symbol}` 允许列表）、NoForecastReason 枚举、8 层风险分层说明、研报模板，供 Skill 运行时引用而非执行。
- 校验脚本层：scripts/ 提供三个独立 CLI 工具，均以 `main(argv)` + `raise SystemExit(main(...))` 暴露，通过 exit code 0/非零 表达 pass/fail，并以 `-` 或文件路径两种方式接收输入；validate_forecast.py 校验 Forecast JSON 必填字段与 horizon_days∈(0,250]、reason 白名单、去重；validate_report.py 强制要求 `# Daily Report` / `## Forecasts` / `## Portfolio` 三段标题并对 NO_FORECAST 一致性做正则检查；canonical_id.py 根据前缀将用户 ticker 猜测为候选 InstrumentId，单候选直接打印，多候选 stderr 列出并返回 exit 2 让 Agent 二次确认。
依赖方向单向：SKILL.md 引用 references/ 和 scripts/，脚本之间互不依赖，仅作为外部子进程被 Skill 编排调用。
