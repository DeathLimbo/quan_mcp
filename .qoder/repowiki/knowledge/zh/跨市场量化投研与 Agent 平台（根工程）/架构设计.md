顶层通过 pyproject.toml 的 hatchling 构建目标 `packages, apps` 将 packages/ 与 apps/ 作为同一可安装包的子包发布；apps/api、apps/worker、apps/quant-read-mcp、apps/quant-admin-mcp、apps/scheduler 共享 packages/common 的 InstrumentId、时间戳字段集与错误/日志约定，并通过可选依赖 `[ml]`、`[backtest]`、`[data-sources]` 按需装配。
- 运行时编排由 `deploy/docker-compose.yml` + `configs/{base,dev}` 分层 YAML 驱动，所有服务共用 Redis/RQ、PostgreSQL、MinIO 等基础设施。
- 数据迁移集中管理于 `sql/migrations`，由 alembic.ini 统一入口执行。
- README.md 定义的“顶层约束”（UTC 时间带 event_time_utc / market_local_time / available_at_utc / as_of_time_utc / ingested_at_utc / calendar_version / rule_version、InstrumentId 唯一键、Point-in-time 查询、一份特征代码、Fail-closed 风控最终授权、MCP/Skill/LLM 禁令）是所有 child 模块必须遵守的跨域契约。
