测试按 pytest 约定组织为三层：
- `tests/unit/`：针对单个包（如 `packages.data_sources.adapters.fake`、`packages.ingestion`、`packages.corporate_actions.adjust`、`packages.backtest.execution`、`packages.features`、`packages.inference.service`、`packages.risk.engine`、`packages.portfolio.builder`、`packages.ledger_paper` 等）的单元测试，每个被测模块一个文件。
- `tests/integration/test_e2e_pipeline.py`：串联真实组件链（FakeMarketDataAdapter → SqlBarSink + SqlBarRepository → corporate-action adjust → FeatureSet.compute → InferenceService.score → build_portfolio → RiskEngine.evaluate → Portfolio.apply_fill → sharpe_ratio），用内存 SQLite 和 `_ProdStubModel` 模拟生产模型注册/晋升门控。
- `tests/fixtures/golden/{cn|us|fund}/*.jsonl`：人类可读的 JSONL 黄金数据集，每行带 `type` 字段（bar/action/order/nav），由 `tests/fixtures/__init__.py::load_scenario` 加载，被 `test_golden_scenarios.py` 驱动执行模型与公司行为调整逻辑。

全局入口 `conftest.py` 仅把仓库根加入 `sys.path`，保证在可编辑安装或 Windows 环境下能直接 `import packages.*`；各子包自带空 `__init__.py` 使 `pytest tests/` 可直接发现用例。
