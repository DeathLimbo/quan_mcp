- 配置文件采用 YAML 分层：`base.yaml` 存放不可变默认值，各环境文件通过 `extends: base.yaml` 叠加覆盖，避免重复声明。
- 所有运行时凭据（数据库、Redis、S3、MLflow）一律通过 `${ENV_VAR:-default}` 形式从环境变量注入，不在源码中硬编码。
- Alembic 迁移全部手写且成对实现 `upgrade()/downgrade()`，禁止 autogenerate，revision 使用语义化短串而非 UUID。
- 迁移脚本命名遵循 `YYYYMMDD_NNNN_<snake_case>.py` 格式，确保按时间顺序线性演进。
