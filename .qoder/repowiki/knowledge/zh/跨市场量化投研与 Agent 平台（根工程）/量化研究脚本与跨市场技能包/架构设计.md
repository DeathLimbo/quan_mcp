模块由两个并列子目录组成：
- `scripts/`：一组独立可执行的 CLI 脚本，每个对应一个里程碑任务（#30/#32/#36/#37），通过 `sys.path.insert(0, ...)` 将仓库根加入导入路径，再按需 `importlib.util.spec_from_file_location` 动态加载 `apps/quant-read-mcp/db_backends.py` 和 `apps/quant-admin-mcp/tools.py`，从而在不安装包的本地环境下驱动 `packages.*` 中的数据集构建、LightGBMTrainer、InMemoryModelRegistry、PaperTradingEngine 等核心库。
- `skills/cross-market-quant-research/`：MCP Skill 描述文件 `SKILL.md` 定义触发条件、绝对规则（禁止捏造数字、fail-closed、PIT 纪律、两市场统一 InstrumentId）、标准工作流（resolve → fetch bars → score → risk_check → report）及可用工具清单；`references/` 存放 instrument-id 格式、风险分层、报告模板等参考文档；`scripts/` 下的 `canonical_id.py`、`validate_forecast.py`、`validate_report.py`、`validate_risk_trace.py` 是供 Agent 在运行时调用的辅助验证器。
依赖方向单向：本模块只消费 `packages.*` 与 `apps/*` 的公开接口，不反向被业务代码引用，属于顶层编排层。
